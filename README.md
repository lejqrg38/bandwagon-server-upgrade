## 搬瓦工怎么升级服务器：后台自助操作步骤

搬瓦工的套餐升级，走的是客户后台的 Upgrade/Downgrade 通道，不需要联系客服，全程自助完成。

**具体操作如下：**

1. 登录搬瓦工账户后台（Services → My Services）
2. 找到你想升级的那台 VPS，点击右侧的管理按钮
3. 在管理页面里找到 **Upgrade/Downgrade** 选项，点击进入
4. 页面会列出当前可以升级的目标套餐，选择你要换的那个

6. 系统自动处理，通常几分钟内生效

升级完建议进 KiwiVM 面板手动重启一次，跑一下 `free -m` 和 `df -h` 确认内存和硬盘已经更新过来了。

## 升级差价怎么算？

很多人担心升级会被多收钱，其实系统计算挺透明的。

升级费用 = (新套餐总费用 − 旧套餐总费用) ÷ 计费周期天数 × 剩余天数

比如你年付了一个 $49.99 的 KVM 套餐，买了三个月后想升级到 $169.99 的 CN2 GIA-E 年付套餐，系统会自动算出剩余 9 个月的差价，你只需要补交那部分，不会按全价重新收。具体金额以结账页面显示为准，不需要自己手算。

升级之后，新的计费周期从升级当天开始重新计算。

## 什么时候该升级，什么时候不用

问自己三个问题：

- **流量每个月都跑满了吗？** 如果经常跑到 90% 以上，升配置或者换更大流量的套餐比较合适
- **内存/CPU 跑满导致服务变慢了吗？** 跑 Docker 容器、搭 AI 应用、部署多个服务的话，KVM 的 1GB 内存很快会见底
- **想换更快的线路机房？** 比如从 KVM 普通线路换到 CN2 GIA-E，这个通过升级套餐可以实现

如果只是偶尔流量告急，其实等下个月重置就好，不用急着升级。反过来，如果你已经在考虑"要不要重新买一台配置更高的 VPS"，先看看升级当前套餐够不够——数据不用迁移，IP 不变，省事很多。

## KiwiVM 面板里的两个升级通道别搞混

搬瓦工有两种"升级"，容易混淆：

**套餐升级（Upgrade/Downgrade）**：这是换套餐，配置变了，需要补差价。本文说的就是这个。

**机房迁移（Migrate to another DC）**：这是把你的 VPS 搬到另一个机房，配置不变，免费，数据保留，IP 会变。如果你只是想换机房试试速度，用这个就行，不用升级套餐。

两个操作入口都在 KiwiVM 面板里，别点错了。
